Stock User Restrictions

Restrict warehouses, stock locations, and picking types per-user

Features & Overview

Fine-grained control over user access rights - restrict users to the locations and picking types that they need

Perfect for multi-warehouse operations, or companies that have separate departments and/or managers per location or picking type

Easy-to-use interface - simply add your desired locations & picking types, and click save!


Overview

This module adds powerful, fine-grained security access controls for warehouses, stock locations, and picking types, all on a per-user basis. It works well for operations that use the Odoo Inventory module heavily, such as eCommerce or sales operations. Access control is configured on the User's settings page, so no custom XML or Python code is needed.


Setup

To configure access rules for a User:

  1. Browse to the Access Control page for the desired User (Settings -> Users -> Access Control)
  2. Enable Restrict Warehouses/Locations
  3. Add the desired Warehouses, Locations, and Picking Types
  4. Locations are computed heirarchially - allowing WH will also enable WH / Stock, WH / Receipts, etc
  5. Ensure that the user has User access on the Inventory application
  6. Save the form, and you're good to go!

Please keep in mind that these access rules apply to ALL stock locations and picking types, including virtual and partner locations. You should test your configuration thoroughly before pushing to a production environment. To ensure that Odoo's double-entry inventory system works correctly, you should enable the following locations for all users:

  • Virtual Locations
  • Partner Locations
  • Any PoS-related routes (e.g. PoS Orders)
